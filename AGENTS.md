# Chrono Merge Tactics Development Rules

## 開発ルール

* Godot 4 + GDScriptで実装する。
* Windows 10/11向けの2Dゲームとして作る。
* ゲームモードはEndlessのみ。
* ストーリー、オンラインランキング、課金、ガチャ、物理演算、複数モード、デイリー、ウィークリー、詰めパズルは実装しない。
* 仕様外の機能を勝手に追加しない。
* UIとゲームロジックを分離する。
* 全てのランダム処理は`RunRng.gd`経由にする。
* 同じseedなら同じブロック順・敵出現になるようにする。
* 1ターン処理は`TurnSystem.gd`に集約する。
* 合成処理は`MergeSystem.gd`に隔離する。
* 発動処理は`ActivationSystem.gd`に隔離する。
* 敵処理は`EnemySystem.gd`に隔離する。
* スコア処理は`ScoreSystem.gd`に隔離する。
* 保存処理は`SaveSystem.gd`に隔離する。
* バランス値は可能な限り`data/*.json`で管理する。
* 画像・音・UI素材は、外部の著作物を使わず、このプロジェクト内で生成・作成する。
* 無断で外部素材をダウンロードしない。
* ゲームが起動し、Endlessを遊べる状態を最優先にする。
* 実装後は必ず起動確認・テスト・README更新を行う。

## 禁止事項

* 物理演算を使わない。
* オンライン通信を入れない。
* サーバー機能を入れない。
* 複雑な属性相性を入れない。
* ブロックを初期4種以上に増やさない。
* 敵を初期3種＋中盤1種以上に増やさない。
* 覚醒Lvを初期実装に入れない。
* Lv5以上のブロックを作らない。
* 全盤面探索型の複雑な合成連鎖を実装しない。
* 長いチュートリアルを入れない。
* 長いストーリー説明を入れない。
