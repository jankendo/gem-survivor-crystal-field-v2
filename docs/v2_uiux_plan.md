# v2 UI/UX Plan

## 方針

現行UIは機能量が多い。v2では「常に見たい情報」「状況で見たい情報」「後で詳しく見たい情報」を分ける。

## HUD

常時表示:

* HP
* レベル
* EXP
* 時間
* 撃破数
* スコア
* 結晶破壊数
* 主力ビルド
* Momentumや短時間ボーナス

状況表示:

* 進化可能
* 回収ドローンREADY
* ボス接近/ボスHP
* 危険地帯
* イベント発生
* シナジー発動

詳細表示:

* 全武器/全パッシブ一覧
* 契約/過充電
* 解放進捗
* デバッグ/性能情報

## メインメニュー

v2では、タイトルから「開始」「キャラクター選択」「解放/強化」「武器・パッシブ管理」「図鑑」「実績」「設定」へ素早く移動できる構造を維持する。今後の改善では、最初に押すべきボタンをより強く、補助機能を控えめにする。

## リザルト

リザルトは以下を上から順に示す。

1. スコアと最高記録
2. 今回の成果
3. v2 Momentumや盛り上がり実績
4. ビルド概要
5. 新規解放/進捗
6. 次の目標
7. 詳細ログ

## 初回導線

長い文章チュートリアルは避ける。初回は3枚以内の短い説明にし、プレイ中は「ジェムを吸う」「強化を選ぶ」「危険地帯は報酬が高い」などを状況通知で補助する。

## 実装方針

* HUD文言生成は`V2HudPresenter.gd`へ寄せる。
* ラン内盛り上がりは`V2MomentumSystem.gd`へ分離する。
* 既存`NotificationLogSystem.gd`は活かす。
* 既存のiOS Safe Area/Touch UIは壊さない。

## Phase 2実装済み

* タイトル画面のボタン階層は`TitleScreenController.gd`から生成する。
* ショップカテゴリと図鑑タブ/フィルタ/ソートは専用Controllerでindexを管理する。
* ラン中HUDにMomentum専用パネルとFeedback Directorバナーを追加した。
* Momentum表示は`MOMENTUM II / 残り / 倍率 / 理由`の短い情報に制限する。
* リザルトは「次のランにつながる画面」として、成果、Momentum、解放、通貨、次の目標の順に整理した。
