# v2 Asset Direction

## 世界観

ダークファンタジーとクリスタル発光を組み合わせる。背景は暗く、プレイヤー、敵、ドロップ、重要UIは明るい発光で識別できるようにする。

## 推奨スタイル

| カテゴリ | スタイル |
| --- | --- |
| キャラクター立ち絵 | やや高品質な2Dイラスト寄り。表情とシルエットを明確にする |
| ゲーム内キャラスプライト | 斜め上視点でも読める高可読デフォルメ |
| 敵 | 形で役割が分かる。雑魚は単純、エリートは発光/角/装甲で差を出す |
| ボス | 大型シルエット、強い発光核、専用色を持つ |
| 武器アイコン | 高級デフォルメ高可読アイコン。32pxでも用途が分かる |
| パッシブアイコン | 装備/紋章/結晶として整理し、武器アイコンと形状を分ける |
| 進化アイコン | ベース武器より発光と密度を上げる |
| フィールドドロップ | 背景上で必ず見える強い輪郭と発光 |
| UI | 近未来クリスタル調。ただし視認性優先で過度な装飾は避ける |

## 色設計

* ベース背景: 黒に近い青紫、深い紺、暗い灰紫
* プレイヤー/味方: シアン、白、薄い金
* 通常敵: 赤紫、濁った緑、暗い橙
* エリート/ボス: 強い赤、紫、金、白い発光核
* 報酬: シアン、金、エメラルド
* 危険: 赤、マゼンタ

## 線と塗り

小さいゲーム内表示では、細すぎる線を使わない。アイコンは太めの外形、強い明暗差、透明背景を基本とする。立ち絵は情報量を増やしてよいが、ゲーム内スプライトとアイコンは可読性を優先する。

## 発光表現

クリスタル発光はv2の主役だが、視認性を落とす広いぼかしは禁止する。中心形状が読める発光、輪郭を残したハイライト、背景と分離する縁取りを使う。

## 頭身

キャラクター立ち絵は3.5から5頭身程度。ゲーム内スプライトは2から3頭身程度。敵は役割優先で頭身を固定しない。

## 見やすさのルール

* 32px表示でカテゴリが分かる。
* 64px表示で個別IDが分かる。
* 背景が暗くても明るくても輪郭が消えない。
* 武器、パッシブ、進化、ドロップは形状言語を分ける。
* UI素材はテキストを邪魔しない。

## Phase 2 Batch 01

Phase 2ではP0垂直スライスとして、ノア、スライム、コウモリ、ゴーレム、5分ボス、魔弾、氷軌道、移動速度、磁力、星砕きの魔弾、星晶の原野、タイトル/パネル/カード/Momentum/ボス警告UIをPNG化した。

素材は外部著作物を使わず、`tools/generate_v2_phase2_assets.py`でプロジェクト内生成した。生成条件と再生成/置換プロンプトは`docs/asset_generation/v2_batch_01.md`と`docs/asset_generation/v2_batch_01.json`に保存する。
