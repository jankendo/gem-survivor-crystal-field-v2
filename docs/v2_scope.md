# Gem Survivor Crystal Field v2 Scope

## 今回入れるもの

### P0: 仕様の正本化

`AGENTS.md`を現行サバイバーv2向けに更新し、READMEの詳細役割をdocsへ分離する。v2のビジョン、範囲、ゲームプレイ柱、UI/UX、アセット方向性、アセットパイプライン、移行計画、変更履歴、テスト計画を作る。

### P1: UI/UX改善の土台

HUDにスコア、結晶破壊、主力ビルド、v2 Momentum状態を出す。既存の装備HUDや通知ログは残し、詳細情報はポーズやリザルトへ寄せる。MainとGameScreenの全面分割は行わないが、表示補助は小さなhelperへ逃がす。

### P2: 中毒性強化の基礎

ラン内完結のMomentumシステムを入れる。連続撃破、進化、全ジェム回収、ボス撃破、ビルドシナジーを短時間の盛り上がりとして扱い、視覚通知と軽いスコア倍率へ反映する。

### P3: アート刷新の基盤

アセットマニフェスト、読み込みhelper、画像生成プロンプト雛形を作る。既存SVGはfallbackとして維持し、PNG/WebPへの将来差し替えを先に定義する。

## 今回入れないもの

* 全アセットの画像生成
* 武器31種の全面作り替え
* `Main.gd`、`GameScreen.gd`、`SurvivorState.gd`、`WeaponSystem.gd`の全面分割
* セーブ形式の破壊的変更
* 新しいオンライン/サーバー/音声機能

## 優先順位

1. 動くこと。
2. 既存の面白さを壊さないこと。
3. v2方針に沿うこと。
4. 将来の差し替えと拡張に耐えること。
5. 見た目と手触りを段階的に改善すること。
