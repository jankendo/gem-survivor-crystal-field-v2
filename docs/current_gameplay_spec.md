# Current Gameplay Specification

この文書を、2026-06-23時点の`Gem Survivor Crystal Field v2`現行ゲームプレイ仕様の正本とする。READMEや旧更新メモに矛盾がある場合は、現在のコード、JSONデータ、テスト、この文書の順に優先する。

## ゲームループ

1. タイトルでキャラクター、祝福、必要なら武器/パッシブON/OFFを選ぶ。
2. ラン開始後、探索フィールドを移動しながら敵を倒す。
3. EXPジェム、宝箱、フィールドドロップ、イベント、ギミックでビルドを伸ばす。
4. レベルアップや宝箱で武器/パッシブ/進化を選ぶ。
5. ボス周期を越えながら、より長く生存しスコアと通貨を伸ばす。
6. リザルトで成果、Momentum、進捗、次の行動を確認する。

## ラン内成長

- EXPジェムでレベルアップする。
- レベルアップ報酬は既存候補プール、無効化設定、進化条件を尊重する。
- 宝箱、フィールド装備、イベント報酬はラン中ビルドを補助する。
- ラン内容に関係する乱数は既存`RunRng.gd`系統を使う。

## 武器/パッシブ上限

- 武器、パッシブ、進化、Over Cap、無効化枠は既存の`data/*.json`とSystem実装を正とする。
- v2 Phase 2では武器数、パッシブ数、敵数、ゲームモード数を増やさない。
- 旧ブロック合成やターン制成長は現行仕様ではない。

## 進化

- 進化は既存`EvolutionSystem.gd`、`CharacterEvolutionSystem.gd`、`data/evolutions.json`、`data/character_evolutions.json`の条件で判定する。
- 武器進化はFeedback Directorのcritical通知対象で、Momentum高段階の発動要因にもなる。
- 進化済み武器はリザルトの上位情報に表示する。

## Momentum

- `V2MomentumSystem.gd`がラン内Momentumを管理する。
- 発動要因は連続撃破、無被弾、ボス撃破、武器進化、全ジェム回収、ビルド相性、イベント成功。
- 表示する状態は発動理由、段階、残り時間、倍率、ラン中最大段階、発動回数、合計発動時間、Momentum由来スコア、主な発動要因。
- 同一短時間の重複イベントは`dedupe_window_seconds`で抑止する。
- 10分相当QAの暫定目標は稼働率8-35%、加重平均倍率1.02-1.10、最大倍率1.20以下、Momentum由来スコア3-12%。
- 通常画面へテレメトリ値は出さない。必要時のみ`V2MomentumTelemetry.gd`でCSV/JSON相当のスナップショットを取る。

## EXP不変条件

- EXP必要量は時間で変動しない。
- `tests/test_exp_invariants.gd`と関連テストを現行仕様の根拠とする。
- 旧READMEに時間補正式が残っている場合、それは旧更新メモ扱いで現行仕様ではない。

## ドロップ残存/復活

- フィールドドロップは原則として残存する。
- 復活や再配置は`FieldDropSystem`、`FieldDropSpawnSystem`、関連JSON/テストを正とする。
- 旧「180秒で消える」説明は現行仕様ではない。

## 全ジェム回収

- 全ジェム回収は`GlobalGemCollectionSystem.gd`系統で一括処理する。
- ジェム1個ごとの通知やMomentum発火は行わない。
- Feedback/Momentumでは1回の全回収を1イベントとして扱う。

## 通常敵と特殊攻撃

- 通常敵は原則として弾、爆発、落下物を出さない。
- 特殊攻撃やボス挙動は既存データと専用テストで制限する。
- 旧「射撃敵が通常弾を出す」説明は現行仕様ではない。

## ボス周期

- ボスは`data/bosses.json`の`minute`を正とする。
- ボス出現、ボス警告、ボス撃破はFeedback Directorのhigh/critical通知対象。
- ボス撃破はMomentum高段階の発動要因で、短時間に同じボスを二重集計しない。

## キャラクター進化

- キャラクター進化は既存進化データと進捗保存に従う。
- 進化通知はFeedback Directorのcritical対象。
- リザルトでは発動有無と進捗を表示する。

## セーブ対象

- セーブは`SaveSystem.gd`の現行形式を維持する。
- Phase 2では既存セーブ形式の破壊的変更をしない。
- 選択キャラクター、祝福、通貨、解放、実績、設定、進捗は既存保存形式に従う。

## Windows/iOS操作

- Windowsではマウスとキーボード操作を維持する。
- iOS相当ではSafe Play Area、横画面、タップ対象サイズ、タッチスクロールを維持する。
- 黒帯やノッチ保護領域は入力対象にしない。

## 無音仕様

- 完全無音を維持する。
- BGM、SE、音声、`AudioStreamPlayer`を追加しない。
- 視覚フィードバックは色、形、短い文言、HUD階層で伝える。
